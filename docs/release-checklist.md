# Release checklist

Статус: обязательный список перед публичным запуском.

## Функциональность

- [x] RSS, SQLite-архив, SSR, SEO и постоянные URL.
- [x] Редакционная админка, премодерация комментариев, реакции и статистика.
- [x] Ручная публикация в Telegram и безопасный импорт в черновик.
- [x] CI: синтаксис и безопасный импорт проверяются на Node.js 20 и 24.
- [ ] Финальный smoke test на временном URL Coolify с реальными production
  переменными, но без публикации в боевой Telegram-канал.

## Privacy и безопасность

- [x] Добавить публичную страницу «О проекте, контакты и конфиденциальность».
- [x] Сделать аналитический HMAC дневным и добавить автоматическое удаление
  устаревшей аналитики по заданному сроку хранения.
- [x] Зафиксировать Node.js 20 как минимальную поддерживаемую production-версию.
- [x] Добавить базовые безопасные HTTP-заголовки. Ограничение публичных
  endpoint дополнительно настраивается на reverse proxy/Coolify.

## Инфраструктура

- [ ] Слить PR, зафиксировать release SHA и дождаться CI на `main`.
- [ ] Создать в Coolify один Node.js 20 service с `npm ci` и `npm start`.
- [ ] Подключить writable persistent volume для SQLite `data/`.
- [ ] Настроить healthcheck `/api/health`.
- [ ] Настроить консистентный backup SQLite во внешнее хранилище и проверить
  восстановление.
- [ ] Проверить RSS, импорт, админку, комментарии и Telegram на временном URL.
- [ ] Переключить DNS только после успешного smoke test; GitHub Pages оставить
  как временный rollback и отключать позднее отдельным решением.

## Ввод владельца в момент cutover

- доступ к Coolify и DNS;
- Anthropic API key и подтверждённый биллинг;
- сильные `ADMIN_PASSWORD` и `REFRESH_TOKEN`;
- Telegram token/chat ID и тестовый канал, если Telegram включается сразу;
- доступ к месту хранения бэкапов и выбранный срок хранения;
- решение: только `finskienovosti.fi` или отдельный редирект с `www`.
