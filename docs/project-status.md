# Журнал решений и статус проекта

Дата обновления: 2026-07-20
Ответственный: CTO / Tech Lead

## Принятые решения

- Рабочая архитектура: Node.js + Express SSR + SQLite. JSON используется только
  для разовой миграции, не в рабочем потоке.
- Публичные статьи имеют постоянные адреса `/news/:slug`. Черновики не попадают
  в ленту, API, sitemap и публичные URL до явной публикации редактором.
- Комментарии публикуются только после модерации. Админка закрыта Basic Auth,
  её POST-действия защищены проверкой Origin.
- Редакция может создавать ручные новости, редактировать материалы, задавать
  метки «Срочно» и «Важно», закреплять новости и отправлять выбранный материал
  в Telegram вручную.
- Аналитика и реакции не хранят IP-адреса или User-Agent в открытом виде:
  используются дневные HMAC-идентификаторы. До production нужны срок хранения
  аналитики и политика конфиденциальности.
- Импорт по ссылке разрешён только через безопасный HTTPS-загрузчик с защитой
  от SSRF и всегда создаёт черновик.

## Текущий статус

Готово:

- RSS-сбор, AI/mock-пересказ, SQLite-архив, SSR, SEO и CI;
- редакционная админ-панель, комментарии, статистика, реакции и Telegram;
- безопасный импорт создаёт черновики в mock-сценариях.

Runtime-блокер устранён: импорт реального `https://example.com/` снова
работает, а закреплённый DNS lookup совместим с режимом `options.all`. CI
прошёл изолированные тесты импортера на Node.js 20 и 24.

До запуска также требуются:

- настройка Coolify с persistent volume для SQLite и резервным копированием;
- DNS, HTTPS, production `SITE_URL` и секреты;
- повторный QA на целевой версии Node.js.

## Условия production-запуска

- Запускать только один экземпляр приложения: RSS cron, SQLite и часть
  антиспам-защит рассчитаны на один процесс.
- Coolify должен использовать Node.js 20, `npm ci`, `npm start`, healthcheck
  `/api/health` и writable persistent volume для `data/finskienovosti.db`.
- Бэкап SQLite должен быть консистентным, с проверенным восстановлением и
  отдельным внешним хранилищем; простого копирования одного DB-файла при WAL
  недостаточно.
- `SITE_URL` должен точно совпадать с публичным HTTPS-origin, иначе нарушатся
  canonical URL, sitemap и CSRF-проверка админки.
- Существующий GitHub Pages обслуживает старую статическую версию. Его нельзя
  отключать до подтверждённой работоспособности Coolify; затем потребуется
  отдельное разрешение владельца на переключение DNS/Pages.

Production hardening завершён: Node.js 20 — минимальная поддерживаемая версия,
аналитика и реакции используют дневной HMAC, записи старше 90 дней очищаются,
а SSR отвечает с базовыми безопасными HTTP-заголовками. Страница `/about`
объясняет источники, модерацию и обработку данных.

Полный обязательный список запуска находится в
[`docs/release-checklist.md`](release-checklist.md). До закрытия этого списка
новые продуктовые функции не добавляются: приоритет — безопасный и обратимый
первый запуск.

## Правило работы

Backend, Frontend, QA и DevOps работают через общий репозиторий. CTO принимает
результаты, фиксирует решения в этом журнале и обращается к владельцу только за
доступами, секретами, необратимыми действиями или финальной приёмкой.
